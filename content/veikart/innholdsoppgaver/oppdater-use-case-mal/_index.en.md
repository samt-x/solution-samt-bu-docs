The current use case template does not clearly enough distinguish between user needs and solution proposals. In several cases (particularly those built on concrete project proposals), the user-centred desired situation and solution hypotheses are mixed together in the same section.

## Proposed change

Add a new section called **«Input on solution choices»** after «Desired situation» in the template:

| Section | Purpose |
|---------|---------|
| Desired situation | User-centred target state – solution-agnostic |
| **Input on solution choices** *(new)* | Preliminary solution hypotheses and possible deliverables |
| Insight work | Source references and knowledge base |
| Affected project goals | Link to overarching goals |

«Input» signals that these are not decided choices, and «solution choices» points toward a decision process rather than finished answers.

## Remaining work

- Review all 22 cases and assess whether «Desired situation» mixes needs and solutions
- Extract solution content into the new section where appropriate
- Update «About this website» / editor guide with a description of the two sections

**Reference:** Already implemented in [case 21 – Choice of further education path](/samt-bu-docs/behov/use-cases/21-valg-utdanningsloep/).
